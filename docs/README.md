# Documentation

This directory is the research handoff layer for FlipDist. It explains the
problem setting, solver architecture, benchmark protocol, retained results, and
current open research limits.

Start here:

- `terminology.md`: plain definitions for terms used in output and benchmarks.
- `references.md`: paper citation and background for the Li-Xia framework.
- `architecture.md`: how the C++ solver maps to the algorithmic structure.
- `reproducible-research.md`: how this repository adapts a reproducible
  research template.
- `benchmarks.md`: maintained commands for sweeps, parity, and comparisons.
- `hard-limit-analysis.md`: current empirical boundary and bottleneck notes.
- `data-artifacts.md`: tracked summaries, ignored outputs, and regeneration
  policy.
- `development.md`: build, test, and contribution workflow.
- `repository-inventory.md`: tracked versus ignored repository surface.
- `research-notes.md`: compact handoff notes for future work.
